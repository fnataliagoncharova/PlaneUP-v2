# Model V2

Минимальная V2-модель описывает производственное ядро без опоры на V1. В центре модели находится единая номенклатура, для конкретной выходной позиции задаётся маршрут, маршрут состоит из шагов, каждый шаг выпускает один результат, использует один или несколько входов и может выполняться на основном или альтернативном оборудовании.

## Таблицы

- `nomenclature` — единый справочник номенклатурных позиций. Для каждой позиции хранится код, наименование, единица измерения и признак активности.
- `processes` — справочник производственных процессов, которые используются в шагах маршрута.
- `routes` — маршрут получения конкретной выходной номенклатуры.
- `route_steps` — шаги маршрута в порядке выполнения. Каждый шаг связан с процессом и имеет один выходной результат с количеством `output_qty`.
- `route_step_inputs` — входы шага с нормой расхода `input_qty`. Если вход есть в общей номенклатуре, он задаётся через `input_nomenclature_id`. Если вход внешний и не заведён в справочник, он задаётся через `external_input_name`.
- `machines` — справочник оборудования.
- `route_step_equipment` — оборудование, допустимое для выполнения шага. Здесь задаются основное или альтернативное оборудование, приоритет выбора и номинальная производительность в `м²/мин` или `м.п./мин`.

## Ключевая логика

- Номенклатура единая для результатов маршрутов и для входов шагов.
- Маршрут описывает способ получения конкретной позиции из `nomenclature`.
- Шаг маршрута всегда имеет один выходной результат.
- Шаг может иметь несколько входов.
- Внешний вход шага задаётся через `external_input_name`.
- Для одного шага можно задать основное и альтернативное оборудование.
- Производительность хранится в `route_step_equipment`.
